Three Modules, One Mission

Each module handles one part of safely operating AI agents. Together, they cover containment, skill security, and ecosystem awareness.

How It Works

From download to running agent — no terminal, no YAML editing, no manual container setup.

1

Install

Download the installer for your platform. One binary, everything included.

2

Setup Wizard

The wizard checks prerequisites, clones components, creates configs, and builds containers — all with real-time progress.

3

Dashboard

Start, stop, and monitor your agent from a manifest-driven GUI. Every command and status badge is declared, not hardcoded.

Six Defense Layers

✓ Container isolation — read-only root, caps dropped, seccomp, noexec, non-root
✓ Network proxy — domain allowlist, API key injection, request logging
✓ Tool policy — deny list filters tools before the LLM sees them
✓ App restrictions — sandbox.mode off, workspaceOnly, elevated disabled
✓ Exec controls — allowlist security, ask always, safeBins whitelist
✓ Hardening config — DM pairing, no persistence, telemetry disabled
Manifest-driven architecture: the Tauri backend contains zero knowledge of what's inside any component. Each component self-describes via component.yml. Drop in any component with a valid manifest and it renders correctly.
