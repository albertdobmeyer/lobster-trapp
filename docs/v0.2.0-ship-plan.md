### Three open findings — must address before tag

| # | Sev | Finding | Blocker for ship? |
|---|---|---|---|
| F11 | HIGH | Soft Shell config doesn't deliver tools (root cause: `web_fetch` etc. have `profiles: []`, not in coding profile) | **Yes** — blocks the GUI ever exposing a shell switcher. Workaround for v0.2.0: hardcode Split Shell in the GUI; don't show the switcher. |
| F14 | MEDIUM | `tool-control.sh --apply` creates a rogue parallel container | **Yes** — even if user uses CLI, it leaves orphan containers. Workaround: document `make soft-shell` as broken on lobster-trapp installs in this release. |
| F13 | LOW | `tool-control.sh` hardcodes pre-v2-perimeter container name | Not a ship blocker but coupled with F14 |

For v0.2.0: ship with Split Shell hardcoded, no shell switcher in GUI, the above findings tracked publicly in the release notes as "v0.3 work."
